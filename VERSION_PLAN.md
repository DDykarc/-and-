# 回本清单 - 版本管理方案

## 一、目标

在小程序中增加「版本信息」和「更新日志」功能，让用户可以看到当前版本号和历史更新内容。

---

## 二、实现方案

### 1. 版本信息文件

新建 `utils/version.js`，集中管理版本号和更新日志：

```js
// utils/version.js
module.exports = {
  version: '1.1.0',  // 与微信开发者工具上传时填写的版本号保持一致
  changelog: [
    {
      version: '1.1.0',
      date: '2026-05-19',
      changes: [
        '新增版本信息页面',
        '优化数据加载速度',
      ]
    },
    {
      version: '1.0.0',
      date: '2026-04-30',
      changes: [
        '小程序上线',
        '支持商品日均成本计算',
        '支持血糖/尿酸/体重记录',
        '支持趋势图表',
      ]
    },
  ]
}
```

> 每次上传新版本时，同步更新此文件的 version 和 changelog。

---

### 2. 新增页面：关于

路径：`pages/about/about`

页面结构：
```
┌─────────────────────────────┐
│  回本清单                    │
│  当前版本：v1.1.0            │
├─────────────────────────────┤
│  更新日志                    │
│  ─────────────────────      │
│  v1.1.0 · 2026-05-19        │
│  · 新增版本信息页面           │
│  · 优化数据加载速度           │
│  ─────────────────────      │
│  v1.0.0 · 2026-04-30        │
│  · 小程序上线                 │
│  · 支持商品日均成本计算        │
│  · 支持血糖/尿酸/体重记录     │
│  · 支持趋势图表              │
└─────────────────────────────┘
```

---

### 3. 入口位置

在 **健康记录 → 设置** 页面底部增加「关于」入口，点击跳转到关于页面。

```
┌─────────────────────┐
│  身高：170 cm        │
│  目标体重：65 kg     │
│  性别：男            │
│  对比天数：7 天       │
│                     │
│  ─────────────      │
│  关于 >              │   ← 新增入口
└─────────────────────┘
```

| 文件 | 操作 | 说明 |
